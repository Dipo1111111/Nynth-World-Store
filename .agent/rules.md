# Agent Rules

## ⚡ MANDATORY: Check Skills Before Every Response
Before doing ANYTHING — writing code, giving advice, or planning — you MUST:
1. Read the user's request
2. Scan your available skills for any that are relevant
3. If a relevant skill exists, apply it and mention which skill you are using
4. Only then proceed with the response

This applies to EVERY request, no exceptions. Do not skip this step even for small tasks.

---

## Before Modifying Anything
- If the request is ambiguous in ANY way, ask clarifying questions before writing a single line of code. Do not assume. Do not proceed.
- List every file you intend to modify and wait for my approval before starting.
- If completing a task requires touching a file I did not mention, STOP and ask me first.

## While Modifying
- Only modify what was explicitly requested. Nothing else.
- Do not refactor, rename, reformat, or "clean up" anything I did not ask about.
- Do not add comments, console.logs, or extra imports unless asked.

## After Modifying
- After every change, verify the modified code does not break anything connected to it.
- If something you touched may have broken a dependency, fix it immediately and tell me what happened — do not wait for me to discover it.
- Never consider a task done until you have confirmed nothing else broke.

## Communication
- If something is unclear: ask, don't guess.
- If you are about to do something beyond the exact scope: stop and ask.
- Never apologise for breaking things. Just fix them and explain what went wrong.
